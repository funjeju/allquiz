# 🚀 Viral Loop & KakaoTalk Integration Spec

## 1. 바이럴 심리 설계: "자랑과 도발"
유저가 퀴즈를 완료한 직후, 단순한 "공유하기" 버튼이 아닌 상대방의 승부욕을 자극하는 시스템을 구축합니다.

### A. 동적 Open Graph 이미지 (Vercel OG)
공유 링크의 썸네일 이미지를 정적 이미지가 아닌, **유저의 실시간 랭킹 데이터가 박힌 커스텀 이미지**로 렌더링합니다.
- **노출 데이터:** 발송자의 닉네임, 지역/연령 랭킹, 그리고 도발적인 AI 카피.
- **이미지 예시 텍스트:** *"노형동 30대 상식 1위 김**님이 당신에게 도전장을 보냈습니다. 님은 꼴찌 예상됨 ㅋ"*

### B. Kakao Link API 템플릿 (Feed Type)
- **Title:** AI가 생성한 도발적 헤드라인 (예: "오늘 K-pop 퀴즈 난이도 미쳤음. 넌 절대 100점 못 맞음")
- **Description:** "현재 김**님의 점수는 950점 (상위 2%). 지금 바로 도전해서 코를 납작하게 해주세요!"
- **Button:** `[내 예상 등수 확인하기]`, `[도전장 수락하기]` (클릭 시 딥링크로 퀴즈 세션 직행)

## 2. 초대 보상 시스템 (Referral Rewards)
- **초대자 (Sender):** 공유한 링크를 통해 지인이 퀴즈를 1회 완료할 때마다 '틀린그림찾기(가챠)' 입장권인 **골든 티켓 1장** 즉시 지급.
- **수락자 (Receiver):** 링크를 통해 첫 접속 시 랭킹 부스트 효과(첫판 한정 점수 1.2배) 제공.

## 3. 기술 구현 흐름 (Data Flow)
1. 유저 A가 퀴즈 완료 후 결과 창에서 [카톡으로 친구 도발하기] 버튼 클릭.
2. Next.js API Route에서 유저 A의 점수와 랭킹을 조합하여 고유한 `share_token` 생성.
3. 카카오 SDK 호출, Vercel OG 엔드포인트(`api/og?user=A&score=950&rank=1`)를 썸네일 이미지 URL로 지정.
4. 카톡 메시지 발송 완료.
5. 유저 B가 카톡 메시지 클릭 -> 딥링크 파라미터(`?ref=A_token`)를 달고 앱 진입.
6. 유저 B가 퀴즈 완료 시 Firestore `viral_logs`에 기록되고 유저 A에게 보상 지급.

① “문제 단위 공유” 플로우 추가
## C. Question-Level Share (핵심 추가)

- 특정 문제 틀렸을 때 즉시 공유 유도
- 메시지 예시:
  "이 문제 맞추면 내가 인정한다 ㅋㅋ"

- 버튼:
  [너 이거 맞출 수 있음?]
② 클릭 후 즉시 퀴즈 시작 (지금보다 더 공격적으로)
- 랜딩 후 설명 없이 바로 문제 노출
- 로딩 없이 1초 내 시작
③ 비교 시스템 추가
- A vs B 점수 비교 UI
- “너는 40점, 친구는 80점” 노출
